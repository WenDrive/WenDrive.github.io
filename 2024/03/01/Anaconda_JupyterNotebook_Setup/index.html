windows环境下Anaconda安装JupyterNotebook以及配置

发表于2024-03-01|更新于2024-04-07|机器学习与人工智能
|阅读量:

注意事项：

  • 有问题或者意见及时留言，每个人的能力都有其边界，但集体的智慧是无限的，因此笔者对每一份建议都很重视
  • 笔者环境信息如下
    • anaconda2.5.3
    • base环境python3.11.7
    • jupyter notebook 7.0.8
    • jupyter lab 4.0.11

问题描述

​ 笔者用的windows系统，安装了anaconda环境，并通过anaconda安装jupyter notebook，但是在web页面上的使用存在诸多不便

  • 默认打开的是当前windows用户的目录
  • 不支持代码提示
  • 查看函数方法
  • 安装中文

更改文件目录

​ 笔者使用的环境如下，windows11，anaconda navigator2.5.3，python使用的是base环境，截止到编写此文章的时候的版本为3.11.7，Jupyter Notebook的版本为7.0.8。

常见的方法有两种：

  • 修改配置脚本并重启
  • 带参数启动

方法一：修改配置文件

​ 笔者用的也是方法一，anaconda多py环境，首先需要先确认切换到使用环境中

环境查看

​ 进入anaconda 命令行界面，默认情况下，windows安装后会存在两个应用，或者可以通过anaconda navigator界面点击进入，笔者这里选择的是cmd的方式，也就是下图中的第一种

​ 使用命令行查看conda指令列表，两种方式均可

​ 切换conda环境

​ 退出conda环境，退出当前conda环境到基础环境

配置文件路径查看

切换到安装jupyter notebook安装的py环境下，生成配置文件，如果生成过也会显示对应的文件路径

打开路径下的文件，并使用文本编辑器进行修改，找到 # c.ServerApp.notebook_dir，删除# 并添加自己想要的路径

关闭并启动jupyter

​ 默认情况下，每次在anaconda界面上点击jupyter的launch的时候，都会默认启动一个新的jupyter实例，暴露一个新的端口

​ 使用Anaconda Prompt进入命令行，执行一下命令查看当前实例和对外暴露的端口

进入对应的页面点击shut down（建议全部关闭），之后在 anaconda navigator界面再次点击launch，新实例会应用到配置好的文件目录下

方法二：带参数启动

该方法适合自己安装的jupyter，anaconda的使用方法一更方便

这样新启动的jupyter实例就会将路径定位到指定的文件夹下

jupyter notebook的扩展需要安装jupyter_contrib_nbextensions，该插件不再支持7.x以上的jupyter notebook

我安装的版本是7.x的版本，在网上查看了很多教程，均无法实现

查阅文章👉【jupyter notebook中插件 nbextensions 安装失败分析与解决方法】，发现该插件已经不支持jupyter notebook7.x以上的版本

jupyter notebook 7.x

​ Jupyter Notebook 7.x 版本（和 JupyterLab 一样，因为从 JupyterLab 3.x 开始，Jupyter Notebook 和 JupyterLab 的后端合并）内置了对代码自动补全的支持。这种支持是通过集成的语言服务器（如 Python 的 Language Server Protocol, LSP）提供的，它能够提供更加智能的代码补全、悬停提示、定义跳转等功能。

​ 默认情况下自动补全是使用tab触发的，如果需要实时触发的话，需要进行简单的设置。
