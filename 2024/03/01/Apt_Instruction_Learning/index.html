简介

​ apt 是用于处理 Debian 及其衍生版（如 Ubuntu）上的软件包的命令行工具。我们可以在多个资源中找到有关 apt 的官方文档和信息。

​ 常见官方文档如下：

  • Debian Wiki APT
  • Ubuntu AptGet Howto(https://help.ubuntu.com/community/AptGet/Howto

Debian Wiki - Debian 官方 Wiki 提供了关于 apt 工具的使用和特性的详尽信息。可以访问 Debian Wiki 来获取有关 apt 的详细说明和教程。

Ubuntu Documentation - Ubuntu 也提供了它的软件包管理工具的详细文档，其中包括 apt 的使用指南。Ubuntu 的文档通常是针对新用户更友好的，包含实际的命令示例

区别

​ 这里简单介绍apt，apt-get以及dpkg

​ dpkg、apt 和 apt-get 是 Linux 系统（尤其是基于 Debian 的系统，如 Ubuntu）中的包管理工具，每个工具都有其特定的用途和功能。下面是它们之间的主要区别：

  1. dpkg：
    • dpkg（Debian package）是一个底层的包管理工具，用于安装、删除和管理 Debian 格式的软件包（.deb 文件）。
    • 它不处理依赖关系，意味着如果一个包依赖于其他包，dpkg 不会自动安装或更新依赖的包，用户需要手动管理这些依赖。
    • 常用命令包括安装 (dpkg -i) 或删除 (dpkg -r) 软件包。
  2. apt-get：
    • apt-get 是一个更高级的包管理工具，可以自动处理包的依赖关系。
    • 它使用 APT（Advanced Packaging Tool）系统的功能，从定义好的软件仓库（repository）下载和安装包，同时解决所有依赖关系。
    • apt-get 提供了包括安装 (apt-get install)、更新 (apt-get update)、升级 (apt-get upgrade) 和清理 (apt-get autoremove) 等功能。
  3. apt：
    • apt 是一个较新的命令行工具，旨在简化 apt-get 和 apt-cache 等工具的使用，提供更友好的界面。
    • apt 提供了大多数日常包管理任务所需的命令，例如安装 (apt install)、删除 (apt remove)、更新软件包列表 (apt update) 和升级软件包 (apt upgrade)。
    • apt 通常更适合终端用户，因为它的输出更易于阅读，命令也更为直观。

总结来说，dpkg 是一个基本的包工具，主要用于.deb文件的管理，而不自动处理依赖。apt-get 和 apt 都能处理依赖关系，并从软件仓库中管理包，但 apt 提供了更现代化的用户体验。对于大多数用户来说，apt 通常是更好的选择，因为它简化了包管理操作。
